# Mathematical Core

This project uses a reputation and dissent model to avoid stagnation.

## Reputation Dynamics

- High-reputation agents are not allowed to remain static forever.
- If progress stalls, reputation decays by a small amount each cycle.
- New or younger agents get a grace period so exploration is not punished too early.

## Innovation Pressure

- Innovation is measured as frontier movement, not mere activity.
- Sponsoring new discoveries can offset reputation decay.
- The system prefers cumulative knowledge growth over repeated status retention.

## Dissenting Hypotheses

- Rejected ideas are not deleted.
- They are stored as dissenting hypotheses for later simulation.
- If an alternative proves better, the system can promote it into the main SOP.

## Operational Meaning

- Use reputation as an incentive, not as a permanent rank.
- Preserve failed or minority ideas as searchable memory.
- Prefer measurable progress, not rhetorical output.
