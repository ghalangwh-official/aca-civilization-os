# Contributing to ACA

Thanks for helping build ACA.

## What We Need First

- ROS 2 launch integration
- safer edge execution
- cloud ingestion robustness
- tests and CI
- documentation improvements

## Before You Submit

- keep changes small and reviewable
- add or update tests when behavior changes
- do not hardcode secrets
- prefer explicit contracts over hidden assumptions
- preserve the repository's memory-first architecture

## Suggested Workflow

1. Open an issue or pick one from the roadmap.
2. Describe the scope clearly.
3. Make a small branch or focused commit.
4. Add tests or a reproducible demo when possible.
5. Submit a pull request with a concise summary.

## Good First Contributions

- improve launch files
- add sample payloads
- improve docs wording
- extend tests
- refine roadmap and labels

## Review Expectations

- code should be readable and deterministic
- safety logic must fail closed
- public configuration should use example files and env vars
- docs should explain why a module exists, not only what it does
